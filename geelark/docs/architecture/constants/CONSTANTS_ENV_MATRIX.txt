SECTION 16: BEST PRACTICES & GUIDELINES
================================================================================

When Using Constants:
  1. Import constants from centralized locations (src/constants/*)
  2. Use UPPER_SNAKE_CASE for constant names
  3. Group related constants in objects
  4. Document units and purposes in comments
  5. Prefer constants over magic numbers
  6. Use type definitions for validation

When Setting Environment Variables:
  1. Never commit credentials to version control
  2. Use .env files (gitignored) for local development
  3. Use environment-specific files (.env.development, etc.)
  4. Document all required environment variables
  5. Provide sensible defaults where possible
  6. Validate environment variables on startup

When Adding New Constants:
  1. Choose appropriate section based on category
  2. Follow UPPER_SNAKE_CASE naming convention
  3. Add JSDoc comment with purpose and usage
  4. Document any related environment variables
  5. Add TypeScript type if applicable
  6. Update this matrix with new entries
  7. Update class documentation if relevant
