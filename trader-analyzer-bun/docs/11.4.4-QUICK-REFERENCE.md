## üìã Module Details

### Core Development (`mlgs-core`)

**Windows:**
1. **Main Dev**: Editor | Dev Server | Tests
2. **Git & Build**: Git Status | Build Commands
3. **Type Check**: TypeScript validation

**Commands:**
- `bun run dev` (auto-started)
- `bun test --watch` (auto-started)
- `bun run typecheck`
- `bun run lint`

---

### Analytics (`mlgs-analytics`)

**Windows:**
1. **Dashboard**: Live trading dashboard
2. **Correlation**: Health check | CLI reference
3. **Metrics**: Performance monitoring | Benchmarks

**Commands:**
- `bun run dashboard`
- `bun run correlation health`
- `bun run correlation graph <eventId>`
- `bun run bench:csv`, `bun run bench:position`

---

### Research (`mlgs-research`)

**Windows:**
1. **Console**: Interactive Bun console (MLGS context)
2. **Research Scripts**: Available scripts | Anomaly detection
3. **Data Analysis**: Query tools | Data exploration

**Commands:**
- `bun run console`
- `bun run covert-steam`
- `bun run scripts/research-*.ts`
- `bun run debug:anomaly`, `bun run debug:graph`

---

### Monitoring (`mlgs-monitoring`)

**Windows:**
1. **Logs**: All logs | Errors/Warnings | Warnings only
2. **Chaos**: Test commands | Test results
3. **Health**: System health | Log metrics

**Commands:**
- `bun run chaos`
- `bun run chaos:kill-layer4`
- `bun run health`
- Log monitoring (auto-refreshing)

## üîß Session Management

```bash
# List all sessions
tmux list-sessions

# Attach to session
tmux attach -t mlgs-core
tmux attach -t mlgs-analytics
tmux attach -t mlgs-research
tmux attach -t mlgs-monitoring

# Kill session
tmux kill-session -t mlgs-core

# Switch sessions (from within tmux)
Ctrl-Space s  # List and switch
Ctrl-Space B  # Choose session
```
