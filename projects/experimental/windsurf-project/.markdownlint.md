# MarkdownLint Configuration

# This configures markdownlint to ignore certain files and disable specific rules
# that are not relevant for this project.

## Files to Ignore
- `reports/**/*` - Auto-generated test reports
- `node_modules/**/*` - Dependencies
- `dist/**/*` - Build output
- `**/*.min.md` - Minified markdown files
- `**/*.generated.md` - Generated documentation

## Disabled Rules
- **MD013**: Line length - Allow long lines for code examples and URLs
- **MD033**: HTML elements - Allow HTML in markdown for enhanced formatting
- **MD041**: First line heading level - Allow flexibility in heading structure

## Configuration
```json
{
  "default": true,
  "MD013": false,
  "MD033": false,
  "MD041": false,
  "ignoreFiles": [
    "reports/**/*",
    "node_modules/**/*", 
    "dist/**/*",
    "**/node_modules/**/*",
    "**/*.min.md",
    "**/*.generated.md"
  ]
}
```

## Usage
Most IDEs and editors will automatically pick up this configuration.
For command line usage:

```bash
# Lint all markdown files except ignored ones
markdownlint "**/*.md"

# Lint specific file
markdownlint README.md

# Fix automatically fixable issues
markdownlint "**/*.md" --fix
```
